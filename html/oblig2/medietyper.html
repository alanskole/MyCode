Medietyper

Medietyper i CSS betyr at stilen kan tilpasses etter hvilken type medieenhet HTML blir vist på, som for eksempel en TV, ulike skjermer, ark, talesynteser etc. Disse kan legges til via et eget CSS stilark og lenkes til HTML på samme måten som den vanlige CSS-filen legges til. Derimot må det legges til en "media=''" innenfor "<link>" taggen som for eksempel "<link rel="stylesheet" href="print.css" media="print">" hvis det er et medietypen print det gjelder. En annen metode er å bruke "media rules" som legges til direkte i hoved-CSS eller HTML som gjøres ved hjelp av "@media" for å spesifisere medietypen reglene skal gjelde for. Grunnet et mer oversiktlig resultat anbefales det å lage separate stilark for ulike medietyper. CSS har ulike medietyper som kan benyttes til ulike formål:

  • "all": denne gjelder for alle enheter, og inkluderer ikke egendefinerte instillinger for spesifikke enheter. Hoved-CSS filen er regnet som "all".
  • "aural": brukt for talesynteser, men har blitt erstattet av medietypen "speech" fra CSS 2.1.
  • "braille": medietype for punktskrift på touchscreen med taktil feedback for synshemmede personer.
  • "embossed": punktskriftprintere for blindeskrift som skriver ut taktile ark.
  • "handheld": er ment for håndholdte enheter med små skjerm. Var en popular medietype før, derimot brukt lite nå for tiden på grunn av responsiv design.
  • "print": brukes for utskrift slik at kun ønsket innhold fra nettsiden printes. Uønskede deler av nettsiden kan skjules slik at unødvendig materiale som er uegnet for utskrift kan ekskluderes fra utskriften, som for eksempel video, meny, animasjon, footer, header. Samtidig kan en rekke andre ting endres for å unngå unødvendig bruk av fargetoner og for en bedre leseopplevelse, som for eksempel endring av font, skriftstørrelse, farge på tekst og bakgrunn, linjehøyde, papirstørrelse etc.
  • "projection": beregnet for visningsmodus, for eksempel projiserte presentasjoner.
  • "screen": stilark primært tiltenkt skjermer på desktop PC, men har i senere år blitt vanlig til både mobil og tablet. Dette er den mest vanlige medietypen siden alt utenom speech og print er ment for visning på skjerm.
  • "speech": erstattet "aural" fra og med CSS 2.1. Sette regler for hvordan skjermlesere leser innhold på skjermen for svaksynte, men benyttes også for lydbøker. Har blitt viktig å implementere grunnet mennesker med svekket synsfunksjoner. Medietypen kan gjøre spesifikke endringer på for eksmepel stemmer, lyder og intonasjoner for å optimalisere lytteopplevelsen siden browserne talesynteser ikke alltid er optimale. Aural box model brukes til å ha pauser mellom lesningen slik at leserne får med seg mest mulig av innholdet. Voice family, voice balance, voice pitch, voice volume i CSS kan endres slik som for eksempel tekst endres av CSS.
  • "tty": for medier hvor hvert tegn har samme bredden, for eksempel gamle TV-skjermer og terminaler.
  • "tv": beregnet for TV'er.

De medietypene som er relevante nå til dags er all, print, screen, speech. I tillegg til disse er det vanlig å bruke "media query". "Media querries" brukes for å speisifere hvilke regler skal gjelde for ulike skjermstørrelser, og tilpasser seg til ulike dimensjoner brukere har på de ulike enhetene sine, mobil, tablet, desktop etc. Det er kun stilen som forandres uten å gjøre endringer i innholdet, i tillegg er denne metoden viktig for search engine optimization. "Mobile-first" er den beste metoden å følge ved webdesing, der man først begynner å designe nettsider til mobilskjermer og skalere opp senere. Dette er fordi mobiler krever mer tilpasning enn datamaskiner som et resultat av mindre skjermstørrelse og berøringsskjerm i stedet for klikk med mus. Ved bruk av "media querries" må "<meta name="viewport" content="width=device-width, initial-scale=1.0">" inkluderes i "<head>" for at det skal fungere.

Dette betyr de ulike leddene i taggen:

  • <meta name="vieport": gir nettleseren instruksjoner om sidens dimensjoner og skalering.
  • width=device-width: sier at sidens bredde skal følge enhetens skjermbredde.
  • initial-scale=1.0>: setter zoom-nivå når siden lastes inn i nettleseren.
