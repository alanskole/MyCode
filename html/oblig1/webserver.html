Webserver

10.09.2019


Først litt om hvordan webserveren ble oppfunnet.
Prosjektet startet i1989 av briten Tim Berners-Lee med mens han jobbet for den sveitsiske fysikk laboratoriet CERN. Internett fantes på den tiden, og ble kun bruk til forskning av forskere. Deling av materiale var derimot vanskelig side nettsider ikke fantes, og det var dette prosjektet skulle løse gjennom teknologien hypertekst. I 1990 kom den første nettsiden http://info.cern.ch på World Wide Web (WWW). Verdens første webserver ble kjørt på en NeXT datamaskin i CERN i 1990. Maskinen hadde en klistrelapp der det var skrevet: "This machine is a server. DO NOT POWER IT DOWN!!" HTTP (HyperText Transfer Protocol) var protokollen som ble brukt.[1]

Hva trengs for å kjøre en webserver?
En webserver trenger i bunn og grunn kun lagring og tilkobling til internett, så teknisk sett er dette ganske simpelt!
Servere kan kjøres på en liten Rasberry PI, eller et rom fullt av store maskiner.

Det er derimot noen krav til maskinene for at servere skal takle høy trafikk uten å kræsje![2]

Hvorfor trengs webservere for publisering på nett?
Alle nettsider man besøker er lagret på webservere, med unntak av sider som kun er lokale, det vil si kun tilgjengelig fra din maskin.
Dette vil si at webservere er nødvendige for å publisere på nett. Egen datamaskin kan også brukes som en webserver, men dette er ikke egnet hvis man har mye trafikk på websiden. Grunnen er at maskinen blir hardt belastet og vil bli ubrukelig til annen bruk, som for eksempel surfing, spilling etc. Belastningen utgjør høyt støynivå og varme, og datakunnskapene til eieren av webserveren må også være langt over gjennomsnittlig. Selv store bedrifter drifter sjeldent egne servere siden det rett og slett ikke lønner seg med tanke på pris.[3] Det er ikke billig å ha antallet nødvendige maskiner, ha eget personale som drifter og vedlikeholder de, pluss at strømregningene blir høye! Derfor er man avhengige av hostere, som drifter webservere man kan abonnere på.

Aksessere en nettside via URL.
Alle nettsider har egne IP-adresser som er besøksadressen, vg.no sine IP-adresser er for eksempel 195.88.54.16 og 195.88.55.16.
Dette betyr i praksis at du kan skrive inn de adressene i nettleseren din og få tilgang til vg på samme måte som vg.no. Heldigvis slipper man å huske på disse kompliserte IP-adressene ved hjelp av URL(UnifiedResourceLocator).
Det flere ledd i en URL. Her er en oversikt over de ulike leddene i https://www.hiof.no/om-oss:

  1. Protokoll, "https", er metoden for sending og mottak av informasjon. Det finnes andre protokoller også, ftp, sftp, http og mailto.
  2. Subdomene, "www", gir tilgang til ulike tjenester eller oppdeling av selve domenet.
  3. Domene, "hiof.no", er navnet man identifiserer domenet med, som egentlig representerer IP-adressen til nettstedet.
  4. Path, "/om-oss", spesifiserer hvilken del av nettstedet man kommer inn på.

Hvem tilbyr webservere?
Det finnes flere tilbydere av webservere, men de mest kjente er Apache, Microsoft IIS, NGINX og Google Web Server (GWS). Såkalte "server stacks" kombinerer flere programvarer som blir brukt for å kjøre webservere på maskiner, der det finnes en rekke ulike kombinasjoner.
De fleste kjører Linux som operativsystem, siden det er et lettvekt og system som belaster maskinene mindre enn andre operativsystemer, også er det gratis i motsetning til Windows. Her er tre ulike som kjøres på tre forskjellige operativsystemer (Linux, Mac OS X og Windows):

  • LAMP, som kjører Linux som operativsystem, Apache server, MySQL eller MariaDB som databasehåndteringssystemer, Perl, PHP, og/eller Python som programmeringsspråk
  • MAMP, som kjører Mac OS X som operativsystem, Apache server, MySQL eller MariaDB som databasehåndteringssystemer, Perl, PHP, og/eller Python som programmeringsspråk.
  • WAMP, som kjører Windows som operativsystem, Apache server, MySQL eller MariaDB som databasehåndteringssystemer, Perl, PHP, og/eller Python som programmeringsspråk.
